TelephoneNumberService
public abstract class AbstractTelephoneNumberService
extends java.lang.Object
implements TelephoneNumberService

Default implementation of the TelephoneNumberService interface. TelephoneNumberService implementation classes should extend this class and override the default implementation as needed. Note that it is not necessary to override the default implementation in order to use different values for the extension delimiters, national prefixes, or international prefixes, since those values are defined by managed plug-in preferences. See the methods in TelephoneNumberService for more information.

Note that AbstractTelephoneNumberService protected methods do not appear in the Sametime SDK Javadoc, but are available for use by customers who provide their own TelephoneNumberService implementation. See the telephone number service sample plug-in in the Sametime SDK Client Telephony Toolkit for more information on these methods.

getTelephoneNumber

public java.lang.String getTelephoneNumber(Address address)
                                    throws TelephonyServiceException
Resolves a telephone number for an Address representing a Sametime person. This method will not be called for Address objects that represent phone numbers or external contacts.

The default implementation retrieves the value of "telephone" attribute in the Sametime directory for the user specified by the Address object. If the Address object also includes an addressUri value, and it doesn't exactly match the value from the directory, the addressUri is returned without further processing. This is because the telephone number service is intended to process numbers retrieved from the directory, not user-specified numbers.

If no directory telephone number was found, the addressUri (if any) from the Address object is returned without further processing. If no addressUri was specified, and the user is someone other than the local user (the caller), the user's location profile is checked for a number (this is typically the number displayed on the user's business card); if found, that number is returned without further processing. If no number can be found, a TelephonyServiceException is thrown.

Assuming the directory telephone number was retrieved successfully, the number is processed as described below and then returned:

1. If any extension delimiters were specified, each delimiter is checked to see if it occurs anywhere within the number, stopping when a match is found. If a match is found, whatever follows the delimiter is assumed to be an extension number, and both the delimiter and extension are removed from the directory number. In addition, if the directory number belongs to someone other than the local user , and the Sametime Computer Phone is being used to make the call, the extension is dialed automatically as soon as the call is connected.

2. If any national dialing prefixes were specified, each prefix is checked to see if it occurs anywhere within the number, stopping when a match is found. If a match is found, the prefix is removed from the number.

3. If any international dialing prefixes were specified, each prefix is checked to see if it occurs at the beginning of the number, stopping when a match is found. If a match is found, the prefix is removed and replaced with the '+' character.

setExtensionDelimiters

public void setExtensionDelimiters(java.lang.String delimiters)
Description copied from interface: TelephoneNumberService
Sets one or more comma-separated extension delimiters, which indicate the presence of an extension in the telephone number. The Sametime Connect client calls this method with the value specified in the managed plug-in preference com.ibm.collaboration.realtime.telephony.ui/extensionDelimiters. The preference value defaults to "ext", but can be changed by the Sametime administrator as needed. See AbstractTelephoneNumberService for more information on how extension delimiters are processed by the default implementation.

Sets one or more comma-separated international dialing prefixes that might be used in a telephone number. The Sametime Connect client calls this method with the value (if any) specified in the managed plug-in preference com.ibm.collaboration.realtime.telephony.ui/internationalPrefixes.
