com.lotus.sametime.core.util

Class STResourceBundle

  • java.lang.Object
    • java.util.ResourceBundle
      • java.util.PropertyResourceBundle
        • com.lotus.sametime.core.util.STResourceBundle

  • public class STResourceBundle
extends java.util.PropertyResourceBundle

    This class represents an extended property bundle. It will fetch property files from an HTTP connection. A locale can be specified to allow for detailed organization of localized properties.

    An optimization has been added that will do a localized check to see if the requested property file exists. This is done by preloading a list of available property files from the server. If requested bundle in not in the list then the http request is not done. This helps reduce the number of hits on the http server.

    A subclass of STResourceBundle could overload the fetch methods to get the data from a source other than http. Using T.Thin back channel or VP server side services would be a logical extension.

    TODO: Look into combining a set of property files into one zip stream. For example all the english sametime property files could be combined into one Sametime_EN_us.zip file. This would reduce both the bytes on the wire and the number of hits to the server.
