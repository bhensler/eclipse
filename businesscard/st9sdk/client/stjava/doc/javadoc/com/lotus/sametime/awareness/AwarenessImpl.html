      • getCurrentStatuses

        public void getCurrentStatuses(java.lang.Integer watchListId)
        Re-send the USERS_STATUS_CHANGED event for all users being watched. This will send out status updates with the latest status for each user that we already have in the WatchList, it does not hit the server again. This is particularly useful for STProxy / mobile - STProxy keeps the WatchList active, but removes the StatusListener when mobile goes into suspend. Because the WatchList remains active, it will contain all current status updates, so when mobile resumes, STProxy can re-add the StatusListener and call this method to get the current status of all users.
