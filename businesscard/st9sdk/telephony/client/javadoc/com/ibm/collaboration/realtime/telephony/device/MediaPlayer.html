Interface MediaPlayer

All Known Implementing Classes:
AbstractMediaPlayer
public interface MediaPlayer

MediaPlayer is used to play audio or video files in a voice or video call. Implementations of this interface may support only audio, only video, or both. A media player can be associated with a specific call or standalone, if supported by the service provider.

Since:
Sametime 7.5.1 (replaces com.ibm.collaboration.realtime.telephony.AudioFilePlayer)

play

void play(java.lang.String filename)
          throws TelephonyServiceException
Plays the specified audio or video file. The file extension is used to determine the file type. If another file or stream is already playing or paused, this automatically stops that file or stream. If this MediaPlayer is associated with a call, the file is played for everyone on the call. If playing a video file, a canvas must first be specified using setCanvas(Canvas), unless the file is to be played in a canvas from the video call. Use getSupportedFileTypes() to determine which file types are supported. Playback starts at the beginning of the file, using the volume scaling specified with setVolumeScaling(double). To play a file starting from somewhere other than the beginning, use play(String, long) instead. To play WAV audio from an input stream, use play(InputStream).

Parameters:
filename - pathname of file to play
Throws:
InvalidParameterException - if the filename is invalid
InvalidOperationException - if a canvas is required but has not been specified
UnsupportedCapabilityException - if the specified file type is not supported
TelephonyServiceException - if some other error occurs

setVolumeScaling

void setVolumeScaling(double scaling)
                      throws TelephonyServiceException
Sets the audio volume scaling used for file playback. Scaling is used to reduce the playback volume relative to the system speaker volume, to control audio output that might otherwise be too loud. The scaling value is from 0-1.0, where 1.0 (the default) is no scaling. If a file is currently playing, the change takes effect immediately, otherwise it takes effect when playback begins or resumes.

Parameters:
scaling - volume scaling, 0 - 1.0 (the default)
Throws:
InvalidParameterException - if the scaling value is invalid
UnsupportedCapabilityException - if the operation is not supported
TelephonyServiceException - if some other error occurs
Since:
Sametime 7.5.1

stop
