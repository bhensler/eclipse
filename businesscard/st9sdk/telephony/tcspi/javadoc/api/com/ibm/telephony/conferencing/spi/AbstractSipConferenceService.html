                                         ConferenceResponseHandler listener)
                                  throws ConferenceException
Description copied from interface: ConferenceServiceRequests
This method indicates a user that should be dialed at the specified end-point. Since the endpoint parameter is a string, it could be a phone number, SIP address, or any other identifier for the location of a user. The service provider should dial the user and then call deliverDialResponse() to indicate success. A failure during processing of this method should result in a call to requestFailed() with the appropriate error.

Parameters:
user - the resolvable user that this call is for. This value may be null.
endPoint - the "callable" end-point such as a phone number or SIP URI
dialMode - a string from the DialModes class that indicates how the user should be dialed
mediaFlag - a flag indicating which media (audio, video, or both) to start
listener - the listener that receives the response
Throws:
ConferenceException - when any exceptional situation occurs that prevents the request from being attempted
