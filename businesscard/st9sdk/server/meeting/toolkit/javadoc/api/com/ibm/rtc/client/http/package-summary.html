Package com.ibm.rtc.client.http Description

Provides generic classes necessary to

  1. Define the most common and basic HTTP services needed by REST APIs.
  2. Abstract the caller from the lower-level toolkit being used.
Note, these classes are not tied to any single product, rather these are intended to work as a re-usable base for any REST API.

HttpServiceWrapper

This class provides the low-level basic HTTP service definition, and encapsulates the lower-level service provider, so that the APIs stay consistent and applications can work with different service providers. See HttpServiceWrapper for details.

ServiceBase

This abstract class provides the basic service definition for common REST API semantics, such login, logout, and helper methods for working with JSON, Cookies, and client state. See ServiceBase for details.

HttpResponseWrapper

This class provides the basic service definition for common HTTP response data, and encapsulates the lower-level service provider, so that the APIs stay consistent and applications can work with different service providers. See HttpResponseWrapper for details.
