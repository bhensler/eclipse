        public HttpConnection(java.lang.String host,
              int port,
              long timeout)
        Deprecated. 
        Creates a new connection to the given server, at the given port. This will use the UrlConnection class provided by the VM, and if this connection is configured to go through a proxy, the connection will be routed through that proxy. On some browsers, the UrlConnection class is known to cause some problems, so you might choose to use our own connection, by calling the constructor that receives a proxy name and port.
        Parameters:
        host - the host to use for HTTP connections. When null will use host from login.
        port - the port to use for HTTP connections
        timeout - the timeout period to use when attempting to connect, in miliseconds

        public HttpConnection(java.net.URL hostUrl,
              long timeout)
        Deprecated. 
        Creates a new connection to the given URL. This will use the UrlConnection class provided by the VM, and if this connection is configured to go through a proxy, the connection will be routed through that proxy. On some browsers, the UrlConnection class is known to cause some problems, so you might choose to use our own connection, by calling the constructor that receives a proxy name and port.
        Parameters:
        hostUrl - Rather than constructing the URL from host, port, and so on, we receive a complete URL. This is especially useful with Reverse Proxies when URL should carry additional information like Server Affinity ID.
        timeout - the timeout period to use when attempting to connect, in miliseconds

        Deprecated. 
        Creates a new connection to the given server, at the given port, through the given proxy. This uses proprietary connection code, which is more stable than some VMs implementations of the standard UrlConnection class. The only drawback is that you must be running as a trusted applet (or an application) for this to work.
        Parameters:
        host - the host to use for HTTP connections
        port - the port to use for HTTP connections
        timeout - the timeout period to use when attempting to connect, in milliseconds
        proxyHost - the proxy to connect to, or null for none
        proxyPort - the proxy port
