Welcome to the Sametime 9.0 Directory and Data Access Toolkit

The Sametime Directory and Data Access Toolkit allows you to customize Sametime to handle the following in your Sametime deployment:

          chat logging - This server-side component allows transcripts of all chats to be stored in a set location, for example, in a specific database. This SPI allows the administrator to determine where and how the chats are stored, by developing a DLL or service progam that implements the specified SPI.

          token authentication - Sametime supports two types of authentication tokens out-of-the-box: 1) proprietary Sametime token; and 2) LTPA token supported by Domino and Websphere. This server side component allows you to customize Sametime for a different kind of token generated in your deployment, by developing a DLL or service program that implements the specified SPI.

          file transfer (antivirus scanning) - This server-side component allows files, sent from one user to another to be scanned for viruses.
This SPI allows the administrator to apply an implementation of antivirus scanning or to develop a DLL or service program that implements the specified SPI.

          user information - This is a Java component which enables users' details to be retrieved from different data resources and be displayed in the business card. This server side component allows you to retrieve users' details from a data storage other than Sametime configured directory, by developing a jar or service program that implements the specified SPI.

The chat logging sample in the Toolkit is currently implemented for the Windows operating system with the Microsoft® Visual Studio .NET 2010 and for Linux OS; However, AIX and iSeries developers may use the same source files in adapting the sample for use in these environments.
