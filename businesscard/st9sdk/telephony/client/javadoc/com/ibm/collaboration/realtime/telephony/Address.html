Interface Address

All Superinterfaces:
java.io.Serializable
public interface Address
extends java.io.Serializable

Address represents the address of a participant or device to be added to a call. When using Address to represent a participant, specify both the address URI and participant ID when using Lotus Sametime Connect telephony services. Other service providers may have different requirements.

Since:
Sametime 7.5

setAddressUri

void setAddressUri(java.lang.String uri)
Sets the address URI of the participant or device to add to a call. For the Lotus Sametime Connect telephony services, which support adding participants only, specify one of the following formats:

voice chat: "voice:" followed by the Sametime contact ID
video chat: "video:" followed by the Sametime contact ID
click-to-call: RFC3966 telephone number URI

Other service providers may have different requirements, for example, an RFC3261 SIP URI.

Parameters:
uri - participant address URI or null
Throws:
InvalidParameterException - if an invalid URI is specified

setIsOffline

void setIsOffline(boolean flag)
Specifies whether this address represents an offline user, for example, an external user or one who isn't currently logged in. The default value is false. The service provider can use this information as needed. For click-to-call, offline users will be required to respond to a prompt before being joined to the call, while online users will not be prompted (depending on the service provider).

Parameters:
flag - true if address represents an offline user, false (the default) otherwise
Since:
Sametime 7.5 CF1
