Class StTlsConfig

  • java.lang.Object
    • com.lotus.sametime.core.util.connection.StTlsConfig

  • public class StTlsConfig
extends java.lang.Object
    Wraps the standard SSLContext class. This class encapsulates a set of TLS configuration settings, and creates an SSLContext based on those settings. One StTlsConfig object is then used for creating many StTlsEngine objects. This class is the Java equivalent of StTlsConfig.cpp in the C++ "ubq" library.

    In addition, this class manages the expiration of cached TLS sessions.

        Creates a new TLS configuration. This API is likely to change in future versions, and applications that use this method will need to make the necessary code changes when upgrading. If this is not acceptable, please use the create(Properties) API instead, as it is less likely to change.
        Parameters:
        providerName - Name of the security provider, null for the default TLS provider.
        trustStoreType - Trust store type, or null to determine by the trustStoreFile extension.
        trustStoreFile - Trust store file, null if no trust store.
        trustStorePassword - Trust store password, may be null if encoded in stash file.
        trustStorePasswordStashFile - Trust store password stash file, null if not stashed.
        trustManagerFactoryAlgorithm - The TrustManagerFactory algorithm, null for default.
        keyStoreType - Key store type, or null to determine by the keyStoreFile extension.
        keyStoreFile - Key store file, null if no key store.
        keyStorePassword - Key store password, may be null if encoded in stash file.
        keyStorePasswordStashFile - Key store password stash file, null if not stashed.
        keyManagerFactoryAlgorithm - The KeyManagerFactory algorithm, null for default.
        keyLabel - Personal certificate label in key store, or null to use any.
        securityLevel - The minimum security level to allow. The applied security level will be pushed higher if there are JVM arguments that impose stronger security.
        minProtocolVersion - The oldest protocol version to allow. Pass 0 to use the recommended minimum version, which is currently TLSv1.
        maxProtocolVersion - The newest protocol version to support. Pass 0 to use the highest version implemented.
        cipherSuites - Comma-separated list of custom cipher suites. null implies default, while "*" implies all implemented cipher suites.
        clientAuth - Client authentication option for server-side engines. null implies ClientAuth.NONE.
        trustedHosts - Comma-separated list of valid peer hosts. Peer verification passes if the host name in the peer certificate matches any of these names. null implies no matching.
        mirrorTrustedHosts - Indicates that the list of acceptable hosts is retrieved from the local personal certificate, in addition to the host names passed in the trustedHosts argument.
        sessionCacheSize - Maximum number of sessions to cache for potential session reuse (-1=none, 0=no limit).
        sessionCacheTime - Maximum time to re-use the same session, in seconds (-1=no reuse, 0=no limit).
        maxSessionAge - Maximum time that an engine will exchange data over the same session before it initiates a new one with the peer, in seconds (0=no limit).
        Returns:
        The newly created configuration object, or null on failure.

      • create

        public static StTlsConfig create(java.util.Properties properties)
        Creates a new TLS configuration according to the properties in the given property bag. The property keys should match the TLS_.. constants defined in this class. Non-matching keys are ignored.
        Parameters:
        properties - The property bag containing TLS settings. The property values are all expected to be of type String, except for the password properties (TLS_TRUST_STORE_PASSWORD/TLS_KEY_STORE_PASSWORD) which may be either a String or a char[]. In case the password property is a char[], the created StTlsConfig object will keep a reference (and not a separate copy) to that character array. This allows the application to erase the password from memory.
        Returns:
        The newly created configuration object, or null on failure.
