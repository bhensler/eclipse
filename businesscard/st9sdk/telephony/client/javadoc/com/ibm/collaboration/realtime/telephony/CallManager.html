Interface CallManager

All Superinterfaces:
Capability
public interface CallManager
extends Capability

CallManager is to start or join a call between two or more parties. Use one of the methods in the Call interface to end the call.

setAudioPort

void setAudioPort(java.lang.String udpPort)
                  throws TelephonyServiceException
Sets the UDP port that will be used to transmit audio streams in a voice chat, if supported by the service provider. The Sametime administrator can set a user policy that specifies which UDP port should be used for voice chat. When that policy is received by the client, this method is called. Note that the port might change during a Sametime session. The default audio port value is 20830. The specified port should be an even number. Note that audio uses two UDP ports, the specified one, and the next sequential one (20831 by default) for error correction.

Parameters:
udpPort - UDP port to use for audio streams
Throws:
UnsupportedCapabilityException - if this operation is not supported
TelephonyServiceException - if the operation fails for some other reason
Since:
Sametime 8.0

setPreferredEndpoint

void setPreferredEndpoint(EndpointDescriptor preferredEndpoint,
                          EndpointDescriptor alternateEndpoint,
                          boolean manualChange)
                          throws TelephonyServiceException
Called when the preferred telephony endpoint for making and receiving calls changes, if supported by the service provider. The preferred endpoint can be changed by the user or as the result of a location change. An optional alternate preferred endpoint can be specified, which is used for incoming telephone calls when the preferred endpoint is not a telephone device. The manualChange parameter indicates whether or not the preferred endpoint was changed manually by the user, using the preferred device drop-down list on the main window toolbar. Once the user manually changes the endpoint, no further automatic changes will be processed, unless the user undoes the manual change.

Parameters:
preferredEndpoint - endpoint descriptor for preferred endpoint
alternateEndpoint - endpoint descriptor for alternate endpoint, null if not specified
manualChange - true if preferred endpoint was manually changed by the user, false otherwise
Throws:
InvalidParameterException - if the preferred endpoint is null or invalid
UnsupportedCapabilityException - if this operation is not supported
TelephonyServiceException - if the operation fails for some other reason
Since:
Sametime 8.0.2

setVideoPort

void setVideoPort(java.lang.String udpPort)
                  throws TelephonyServiceException
Sets the UDP port that will be used to transmit video streams in a video call, if supported by the service provider. The Sametime administrator can set a user policy that specifies which UDP port should be used for video calls. When that policy is received by the client, this method is called. Note that the port might change during a Sametime session. The default video port value is 20832. The specified port should be an even number. Note that video uses two UDP ports, the specified one, and the next sequential one (20833 by default) for error correction. To set the UDP port for audio streams, use setAudioPort(String).

Parameters:
udpPort - UDP port to use for video streams

startUiCall

void startUiCall(com.ibm.collaboration.realtime.people.Person[] persons,
                 java.lang.String callType)
                 throws TelephonyServiceException
Initiates an audio, video, or telephone call to the specified participant(s), via the Sametime Connect user interface, as if the user initiated the call using one of the UI call actions. The participants are specified as an array of Person objects; it is not necessary to include an entry for the caller. A call type can be specified if needed, but the recommended approach is to specify a null call type, which will cause the user's preferred device and other preferences to determine the call type. If a call type is specified, and it's not compatible with the current preferred device, the preferred device is changed to a compatible device before the call is placed.

Unlike startCall(List, CallOptions, CallListener), this method does not return a Call object or provide the application with any control over the call; this method starts a call in the UI, which the user controls as if he or she started the call directly. This method is intended for external plug-ins or applications that provide their own separate UI with actions to start a Sametime Connect call.

Parameters:
persons - array of Person objects for participants to call
callType - one of the CALL_TYPE_xxx values defined in this class, or null to use the default call type
Throws:
InvalidParameterException - if either parameter is invalid
TelephonyServiceException - if the operation fails for some other reason

startUiCall

void startUiCall(java.lang.String number)
                 throws TelephonyServiceException
Initiates a telephone call to the specified telephone number, via the Sametime Connect user interface, as if the user initiated the call using one of the UI call actions. If the current preferred device is not a telephone device, the preferred device is changed to the first available telephone device before the call is placed. See startUiCall(Person[], String) for additional information.

Starting with Sametime 8.5.2 IFR 1, the telephone number can end with an optional passcode with the format ";postd=nnnnn", for example "1-555-666-7777;postd=1234#". The passcode ("1234#" in this example) is dialed automatically when the call is connected, but only when the Sametime softphone is used for the call. The passcode can contain digits, '#', '*' and 'p', where 'p' inserts a brief pause.

Parameters:
number - telephone number to call
Throws:
InvalidParameterException - if the phone number is invalid
TelephonyServiceException - if the operation fails for some other reason
Since:
Sametime 8.5
