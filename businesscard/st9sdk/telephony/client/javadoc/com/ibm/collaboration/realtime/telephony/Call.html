Interface Call

All Superinterfaces:
Capability
public interface Call
extends Capability

Call represents a voice or voice and video call between two or more parties. To start or join a call, use one of the CallManager methods.

acceptIncomingCall

void acceptIncomingCall()
                        throws TelephonyServiceException
Accepts an incoming call in response to an IncomingCallEvent, without connecting to the call, if supported by the service provider. Use this method rather than CallManager.acceptIncomingCall(String, CallListener) for Call objects created using CallManager.createCall(String). To make sure you receive all call events, add a call listener using addListener(CallListener) prior to calling this method. To accept the call and connect to it immediately, use connectIncomingCall(CallOptions) instead.

Throws:
UnsupportedCapabilityException - if this operation is not supported
TelephonyServiceException - if the operation fails for some other reason
Since:
Sametime 8.5.1

disconnectCall

void disconnectCall()
                    throws TelephonyServiceException
Disconnects the current user from this call without ending the call for other participants, but continues to receive call events for the call, if supported by the service provider. Use connectCall(CallOptions) to reconnect to the call. To disconnect from the call and deactivate the Call object, so that no further call events are received, use leaveCall() instead. To stop receiving events for a disconnected call and release any associated resources, call endCall() or leaveCall().

Throws:
InactiveCallException - if the call is not active
UnsupportedCapabilityException - if this operation is not supported
TelephonyServiceException - if the operation fails for some other reason
Since:
Sametime 8.5

endCall

void endCall()
             throws TelephonyServiceException
Ends this call. If the current user is the call moderator, the call is ended and all participants are disconnected, otherwise only the current user is disconnected. If the call is active but disconnected, releases resources associated with the call and stops receiving call events.

                   throws TelephonyServiceException
Sends a DTMF (dual-tone multifrequency) event representing a key on a telephone keypad, if supported by the service provider.
