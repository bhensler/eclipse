Class ServiceBase

java.lang.Object
  com.ibm.rtc.client.http.ServiceBase
Direct Known Subclasses:
RealTimeSession, RoomServices
public abstract class ServiceBase
extends Object

This class is a common base class for concrete classes, including RealTimeSession and RoomServices, and encapsulates low-level services for HTTP transactions and state management. Furthermore, this class provides specific support for logging in and managing security tokens and session timeout control.

The structural pattern allows the low-level HTTP stack to initialized once, and shared between objects. One possible use is shown below, in which case one subclass (ex: RoomServices), is created, then a user is logged-in, then a separate object is created (ex: RealTimeSession) thus sharing the common HTTP stack. 

 
 RoomServices roomServices = new RoomServices("http://server.mycompany.com:80");
 roomServices.login("user@company.com","password");
 
 RealTimeSession rtSession = new RealTimeSession(roomServices,);

public long login(String userName,
                  String password)
           throws LoginException,
                  IOException
Performs a user login, and returns the LTPA token time to live value, which is the amount of before the user must login again. It is recommended, that the user of the API re-login within 2 minutes of the Token's timeout.

Parameters:
userName - The user's identity in userId format, like user@company.com, this user must be in a federated repository, such as LDAP, or flat file.
password - The user's password.
Returns:
The seconds remaining before the user has to re-login. This is directly tied to the server's configuration for the timeout value for SSO/LTPATokens. Typically, the user of the API must re-login within 2 minutes of the Token's timeout.
Throws:
LoginException - Thrown if login fails for some reason. Typically, failure is because of incorrect username/password.
IOException - thrown on the occurrence of communication errors, such as a bad host name, port, or network failure.
