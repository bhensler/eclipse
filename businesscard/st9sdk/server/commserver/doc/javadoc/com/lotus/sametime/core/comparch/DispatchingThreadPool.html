Class DispatchingThreadPool

  • java.lang.Object
    • com.lotus.sametime.core.comparch.DispatchingThreadPool

  • public class DispatchingThreadPool
extends java.lang.Object
    A message-dispatching-thread pool. Normally, each session is associated with a dispatching thread. A dispatching thread pool can be used when many sessions exist, so that they share a pool of dispatching threads. This should probably only be used by server applications using the LightLogin Service to multiplex many sessions. Other apps should not use this feature.

      • DispatchingThreadPool

        public DispatchingThreadPool(int maxThreads,
                     int threadTimeout)
        Create a new pool, with the given limit on the number of threads that will be created, and the given thread timeout. The pool will create new threads as needed until the maximum number is reached, and it will clean up threads when they are idle for mor than the given timeout period, assuming that they are not needed.
        Parameters:
        maxThreads - The maximum number of threads that this pool is allowed to create. use Integer.MAX_VALUE for no limit.
        threadTimeout - The thread timeout, in seconds. -1 means threads do not clean up until the last dispatcher associated with the pool is stopped.
