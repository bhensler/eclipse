CONNECTION_PROPERTY

static final java.lang.String CONNECTION_PROPERTY
Indicates the connectivity state of a particular user. Read only A value of "disconnected" indicates that this user is not currently connected to the conference. A value of "connecting" indicates the service provider know this user is connecting to the conference. For example this will happen when the service provider has successfully dialed a user but they have not yet connected. A value of "connected" indicates the user has made a connection to the conference and can now participate Default: User.DISCONNECTED Type is expected to be APPLICATION_PUBLIC_PROPERTY_TYPE

unregister

void unregister(ConferenceResponseHandler listener)
                throws ConferenceException
This method provides a cleanup mechanism for registered users. This does not imply that a conference will be terminated if its owner is unregistered. An unregistered user will be removed from the conference access list and disconnected from the conference if active. Successful request: listener.unregisterUser is called Event contents: User object from the request. Failed Request: listener.requestFailed with appropriate error code Possible error codes:
