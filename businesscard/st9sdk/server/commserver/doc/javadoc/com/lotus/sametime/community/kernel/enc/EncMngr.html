      • createRequest

        public static int createRequest(EncData encData)
        Get the minimum encryption level from the encryption data object, and fill it's parameter list with possible ciphers matching that encryption level. This function is called on channel creation.
        Parameters:
        encData - The encryption data to work on. The minimum encryption level is extracted from this object, and the possible methods are also returned in this objct.
        Returns:
        An error code. Also, if no error has occured, encData.paramsList holds possible encryption parameters to use.

      • localizeRequest

        public static int localizeRequest(EncLevel requestedLevel,
                  EncData encData)
        Find an encryption method matching the requested encryption level in the list of methods offered by the other side. If such a method was found, localize the parameters for it. (I.e. create all data that is needed for us to talk to the other side). This function is called after a channel was accepted.
        Parameters:
        requestedLevel - The encryption level this side requested.
        encData - The encryption data object we got from the other side.
        Returns:
        An error code. Also, if the operation was successfull, encData.remoteParams holds the params for the preferred encryption method.

      • localizeReply

        public static int localizeReply(EncLevel originalLevel,
                EncData encData)
                         throws java.io.IOException
        This function is called on the side of the creator of the channel, after the other side has accepted it, to make sure that the selected method matches the requested level, and to initialize the method.
        Parameters:
        originalLevel - The encryption level requested for this channel When it was first created.
        encData - The encryption data object to work with.
        Returns:
        An error code. ENC_ERR_CODE_OK if the method was initialized. Also, if the function was successfull, the remoteParams and localParams fields are set correctly.
        Throws:
        java.io.IOException - If something went wrong in reading the remote data.
