liveNameResolve

public void liveNameResolve(java.lang.String[] keys)
Resolve a list of email addresses or similar unique keys. Behavior in ST is that only unique matches are returned. To get multiple matches per key use directoryResolve. There is no need to call setWatch() on results because a watch is set implicitly on all results. Results are returned in future asynchronous callbacks using the onPersonUpdate call. The keys in this case can be anything that might resolve. If the resolve does not produce a single result, then it will fail.

Parameters:
keys - the array of ids or names to look up
See Also:
addSametimeCallback(JSTCallback), JSTCallback

directoryResolve

public void directoryResolve(java.lang.String key)
Resolve a single search term which may have multiple matches. Results are returned in future asynchronous callbacks using onDirectoryResolve. Directory resolve will only resolve the name, and will not put a watch on any resolved user. If multiple results are found based on the resolve, then all of the results will be returned in the callback.

Parameters:
key - the search id or name used for look up
See Also:
addSametimeCallback(JSTCallback), JSTCallback

getGroups

public void getGroups(java.lang.String groupType)
Returns the array of group names used in the local client buddy list. Hierarchical groups are delimited with the string "$NESTED". Use the constant MicrobrokerConstants.nestedDelimiter. The result of the getGroups call is returned using the callback method onGetGroups

Parameters:
