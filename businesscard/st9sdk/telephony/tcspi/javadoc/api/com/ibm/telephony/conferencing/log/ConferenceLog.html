com.ibm.telephony.conferencing.log
Class ConferenceLog

java.lang.Object
  com.ibm.telephony.conferencing.log.ConferenceLog

public class ConferenceLog
extends java.lang.Object

ConferenceLog is a helper class associated with the Conference Logging that allows us to wrap log entries associated with a given call inside a wrapper indicating the logId. This allows us to process the Conference log file and look at all entries for a given conference call. To create the ConferenceLogger, use Logger.getLogger(ConferenceLog.LOGGER_NAME); To use the ConferenceLogger, add one of the following on as an extra parameter to the end of a logger entry:

  • ConferenceLog.formatArg(String logId)
  • ConferenceLog.formatArg(Conference conference)
For exiting method, use ConferenceLog.formatResult(String logId, Object result) or ConferenceLog.formatResult(String logId) if no result being passed. Also available are ConferenceLog.formatResult(Conference conference, Object result) and ConferenceLog.formatResult(Conference conference) The logger is created with a special filter and formatter. For non-throwable cases, it looks for an entry as the last parameter of the LogRecord of the form placeid=... It removes this parameter and uses it to create the wrapper in the Conference log file. For throwable cases, it looks for a Throwable of type ConferenceLog.ThrowableWithId, removes the Id and formats with the Throwable The Conference log file should be in the regular log directory with the name conference.log. If it it can't find the log directory, it will be put in the temp directory.
