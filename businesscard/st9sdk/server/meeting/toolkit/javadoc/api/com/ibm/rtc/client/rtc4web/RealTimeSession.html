Class representing the RealtimeSession. Provides the entry point into all real-time operations controlling session state. A real-time session can be created from a RoomServices object or by passing in a room Id. If a RealTimeSession is created by passing in a roomId, login is required first before joining a room.

Example: 

 RealTimeSession rtSession = new RealTimeSession("http://server.mycompany.com:80",<sessionid>,"userA");
 rtSession.login("userA@company.com","password");
 rtSession.join();

RealTimeSession

public RealTimeSession(ServiceBase base,
                       String sid,
                       boolean useAsyncUpdate)
Creates a RealTimeSession from an existing connection stack to the server. Uses all the primed info from another instance. No need to login again to the server if this constructor is used. Also provides a way to specify if a connection pool should be used for the long poll to the server. This is important for scaling and performance.

Example: 

 RoomServices roomServices = new RoomServices("http://server.mycompany.com:80");
 roomServices.login("user@company.com","password");
 RealTimeSession rtSession = new RealTimeSession(roomServices,<sessionid>,false);
 rtSession.join();

Parameters:
base - Existing instance of type ServiceBase. This can be a RoomServices object.
sid - The session id
useAsyncUpdate - True if a connection pool should be used for the long poll to the server. False if a thread per session is to be used.
