PLACE_BEING_DISPOSED_PROPERTY

static final java.lang.String PLACE_BEING_DISPOSED_PROPERTY
Conference property indicating that a place is being disposed. It should contain a timestamp so we can, just in case, remove it if it sits too long. This property is set when an MCU goes down and we want to mark the place so that subsequent calls will fail quickly until the place is cleaned up. It takes time to clean up all calls on an MCU so until this completes, we want calls on a place associated with that MCU to fail.

Since:
Sametime 8.5.2

VIDEO_IN_USE_PROPERTY

static final java.lang.String VIDEO_IN_USE_PROPERTY
Conference property indicating whether external video is in use for a given call. When the client detects this call property with the value true, it will adjust the call window title to reflect that the call is a video call. This optional property allows TCSPI implementations, if they wish to do so, to indicate the use of external video that the client would not be aware of otherwise. This property can be published at any point after the call begins.

registerUser

void registerUser(java.lang.String userName,
                  java.lang.String userRole,
                  java.lang.String userPin,
                  ConferenceResponseHandler listener)
This method is a required call to setup the service provider for all subsequent actions for users. The user name is used to identify a user as a resource user and will return a unique user object. This user can then be used to provide user based conference control. For any single user name registerUser may be called multiple times with out affecting state. However, once the initial role is set it is expected that User.setProperty to change a users role after registration. Successful request: listener.registerUser is called Event contents: original name, role, and a User object scoped to the conference. Failed Request: listener.requestFailed with appropriate error code Possible error codes:

Parameters:
userName - uniquely identifies the user.
userRole - the initial role of the user
userPin - the pin code if any
listener - the listener that receives the response.
