public class SummaryService
extends Object

The summary services allow the caller to gather data about events that have occurred in a room. The services start with coarsely grained room usage API, which provides time stamps related to exactly when a room has been active. Next, there is a service for finer grained and specific events about users joining, leaving, and actions within the room. Finally, there is a service to generate Meeting Summary Reports in different formats (ex: pdf, rtf, html).
While there are no hard rules about how to use these services, they were designed to be called progressively, as follows:

  1. Room Usage: The caller starts by getting a list of usage dates (when the room was used), based on roomId
  2. Room Events: Then, the caller can zoom-in on a specific usage, and get more distinct events for a given time range:
  3. Room Reports: Finally, based on dateTimes established via the above calls, a finished report is available in different formats and different locales

public List<EventInfo> getRoomEvents(String roomId,
                                     Date startDateUTC,
                                     Date endDateUTC)
                              throws IOException,
                                     SummaryException
Returns the list of EventInfo objects representing fine grained meeting events classified an EventInfo.EventType. This enables the caller to determine when, and what type of activity has occurred in the room. The list of event is in chronological order. In order to access this service, the caller must be joined to the room, or mapped to the admin role on the server.

Parameters:
roomId - The id of the room.
startDateUTC - A required Date object representing the start of a time span to list events.
endDateUTC - An optional Date object representing the end of a time span, passing null keeps the time span unbound and will return the most recent events for the current moment.
Returns:
A list of EventInfo objects representing meeting events.
Throws:
IOException - On communication errors
SummaryException - If there are parameters or access issues.
