Package com.ibm.rtc.client.meeting.room Description

Provides the classes necessary to create and manage meeting rooms on the Sametime Meeting Server.

RoomServices

This class provides the jump-off point for all room related actions. All room actions require login and authentication. Some operations that can be performed from this class are:
  • Creating a room
  • Fetching the caller's rooms
  • Fetching rooms by owner
  • Searching for a room
See RoomServices for more information.

AdminRoomServices

A sub-class of RoomServices, it provides capabilities specific to administrators in addition to other basic room services. Some of the operations it provides are:
  • Creating rooms on behalf of other users. This is useful when users do not have permissions to create rooms themselves.
  • Deleting a room belonging to another user. An administrator has privileges to delete any room on the server.
  • Deleting all the rooms belonging to a user.
If administrator operations are not required, please use RoomServices

MeetingRoom

This class represents the meeting room and all its possible attributes. It is passed into RoomServices and returned as the result of operations as well.

QueryControl

This class enables callers to control the size and scope of a search query and provides methods for ordering/sorting the results. An immutable instance of this class can be used to get the standard search parameters. This class is not thread safe.

QueryResults

This class encapsulates the results of a search. It contains accessors to retrieve the search results, set count, starting index etc...

RoomException

RoomException is thrown when an unexpected error condition is encountered in the usage of the REST API.Documents all possible error codes and descriptions possible from the use of the Sametime Meeting Server REST API.
