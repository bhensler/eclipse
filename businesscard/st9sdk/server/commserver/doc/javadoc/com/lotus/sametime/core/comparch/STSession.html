      • STSession

        public STSession(java.lang.String name,
         DispatchingThreadPool pool)
          throws DuplicateObjectException
        Creates a new STSession and adds it to the list of sessions. The session's message queue will use the given thread pool for dispatching objects. Normally, each session is associated with a dispatching thread. A dispatching thread pool can be used when many sessions exist, so that they share a pool of dispatching threads. This should probably only be used by server applications using the LightLogin Service to multiplex many sessions. Other apps should not use this feature.
        Parameters:
        name - The name of this STSession. Has to be unique to identify this session.
        pool - The thread pool to use for dispatching objects. If null, a new, unlimited, pool will be created.
        Throws:
        DuplicateObjectException - If a session with the same name is already registered in this virtual machine.
