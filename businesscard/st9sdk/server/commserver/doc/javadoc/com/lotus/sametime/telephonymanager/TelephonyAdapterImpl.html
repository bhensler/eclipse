      • initiate

        public void initiate(ClusterManagerListener listener,
            java.lang.String stServerName,
            int port,
            java.util.Properties taProperties,
            java.util.Properties mappingProperties,
            java.util.Properties crProperties)
        Perform Initialization steps : load login manager , connect to all servers and load the resolver component. When this process is over a notification will be sent to the listener There are 3 options for initiation parameters: 1. Get all configuration paramaters as properties object. In this case we skip the file loading and ignore the stServerName and port if provided at this method. This option will take place in case the taProperties or crProperties are not null. If one of the 1 or 2 of the propertiers file is null, the relevante component will use its default values. 2: Load all from files. In this case the stServer is null, the port is 0 and the 3 provided properties objects are null. 3: Load all from files, and ovveride the Sametime server name and port. In this case the stServer is not null and the port is not 0. The 3 provided properties objects are null.
        Parameters:
        listener - - the ClusterManagerListener listener
        stServer - - the Sametime server domain name
        port - - the Sametime server port value
        taProperties - - The TelephonyAdapter configuration propewrties.
        mappingProperties - - The TelephonyAdapter mapping propewrties.
        crProperties - - The CachedResolve configuration propewrties.
